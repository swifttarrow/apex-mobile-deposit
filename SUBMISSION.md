# Checkstream Submission

## Summary

A complete Go backend implementation of a mobile check deposit system for Apex Mobile Deposit.

## What's Implemented

### Core Features
- Transfer state machine (8 states, all valid transitions enforced)
- Vendor IQA stub with 7+ deterministic scenarios
- Full deposit submission flow with business rules
- Operator review workflow (queue, approve, reject)
- EOD settlement batching (6:30 PM CT cutoff)
- Return/reversal with $30 fee and ledger reversal
- Idempotency middleware (X-Idempotency-Key)
- Ledger view and account balance endpoints

### Test Coverage
- Unit tests: state machine, vendor stub, funding rules, ledger
- Integration tests: all 10 scenarios end-to-end

## Running

```bash
go mod tidy
make test    # all tests pass
make build   # produces bin/checkstream
make dev     # start server on :8080
```
